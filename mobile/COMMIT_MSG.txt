Mobile v1.30: real artist from source metadata (stop hallucinating artists)

Fixes the library guessing artists from filenames (e.g. "Hook Up Song" becoming an
artist). Now the REAL artist is captured from the source metadata at download time.

- Every download now passes --write-info-json; after the file lands, Downloader reads
  the sidecar and resolves the artist as: music artist → creator → channel → uploader
  (stripping "- Topic"/VEVO, taking the primary of "A, B"), then stores it in the new
  ArtistStore (artist_meta.json) and deletes the sidecar (folder stays clean). Works
  for YouTube, TikTok, Instagram, etc. — same code path for single / bulk / channel /
  Discover / Assistant downloads.
- MediaMeta.artist() hierarchy is now metadata-first: ArtistStore (captured truth) →
  embedded ID3/media tag → "Artist - Title" filename → Unknown. AI stays the optional
  manual backup (Title clean-up), never the primary source.
- Title clean-up renames now carry the captured artist (ArtistStore.move) so grouping
  stays correct after a rename.

Why not the YouTube Data API for this: yt-dlp's own extraction already returns richer
per-video metadata (the music artist/track/album fields the Data API doesn't expose),
for free, on every platform — so it's the better source of truth here.

Note: applies to NEW downloads; existing files keep using their embedded tag/filename
(or the AI Title clean-up tool).
