Mobile v1.27: Discover key embedded (invisible) + personalized shelves (Picks/More-from-artist)

- YouTube API key is now embedded at build time (secret.properties -> BuildConfig)
  and fully invisible: no in-app key dialog, no settings entry, not stored in prefs,
  not editable or copyable. Discover just works. Removed DiscoverKeyDialog and the
  prefs-based key override; Discover.apiKey() reads BuildConfig only. (The key lives
  only in gitignored secret.properties — never committed.)
- Personalized shelves: the app now learns your most-downloaded artists (top 3 by
  library count) and builds "✨ Picks for you" (top artist, by relevance) and
  "🎤 More from <artist>" rows for the next ones — alongside Trending in Kenya /
  Music / Worldwide.
- Shelves use LazyRow (thumbnails load lazily as you scroll) for smoothness.
- Quota-friendly: search results (100 units each) now cache for 24h; trending 6h.
