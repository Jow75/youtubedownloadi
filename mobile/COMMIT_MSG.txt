Mobile v1.28: Discover search — download-first mini-YouTube (videos + channels)

- Discover.searchMixed(): one YouTube search call (no type filter) returns BOTH
  videos and channels; parsed into SearchResults(videos, channels). Cached 24h.
- Discover search bar: type a query → search on submit (quota-safe, not as-you-type).
  - Video results → a tappable list; tap to queue the download (MP3/MP4 toggle).
  - Channel results → circular avatars; tap a channel to OPEN IT IN THE CHANNEL TAB,
    which auto-scans it (yt-dlp, no API quota) → reuse AI category filter (Music/Live/
    Interview/…), official-only, multi-select, per-video format, Download chosen.
  This makes Discover a download-first browser: find an artist → browse their whole
  catalogue → bulk-grab what you want, never opening YouTube.
- Channel auto-scan hook: ChannelUi.requestScan + ChannelBody.startScan() extracted
  and fired via LaunchedEffect when opened from search.

Architecture note: API = discovery (search, 100 units, cached); yt-dlp = the heavy
browsing/category/download engine (free). The two together = the goal.
