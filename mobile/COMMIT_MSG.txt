Mobile v1.31: remove Smart Search, fix Discover download badge, add AI artist alias-merge

- Removed Smart Search (Library → Tools) and deleted SearchIndex.kt. It built an AI
  embedding index (NVIDIA calls, embeds.json that grew stale — "Indexed 424 of 361")
  for a "search by meaning" use almost nobody needs, while the Library already has
  instant name search. Leaner, faster, less RAM/clutter.
- Discover download badge is now persistent and accurate (tied to the real queue):
  green ✓ = in your library · purple spinner = downloading (stays until it finishes)
  · arrow = available. Tapping is disabled while downloading or already-owned, so you
  can't double-download. (DlTask gained a url; Downloads.isActive(url).)
- AI alias-merge for cross-platform artists: Artists → "Merge aliases (AI)" clusters
  the SAME artist written differently ("Diamond Platnumz" / "diamondplatnumz" /
  "… - Topic") into one canonical name. Ai.mergeArtists + ArtistAlias store; MediaMeta
  now canonicalizes on top of its metadata hierarchy (AI stays the backup).
