Mobile v1.24: assistant-bubble fix, mini-player prev, video A/V sync, Home->category nav, liveliness, snappier nav

Phase J — pre-release polish & fixes:
- Assistant: EVERY completed-download bubble stays tappable permanently (was only
  the newest). Keyed off the saved path, not File.exists() at compose time; on tap
  it resolves the file robustly (exact path, else by filename in the Library) so
  renames/moves never break older results. Bubbles now show cover artwork too.
- Mini-player: added the Previous button (Prev / Play-Pause / Next / Close). Full
  player keeps all controls (speed/seek/repeat/shuffle/timer).
- Video A/V sync: downloads now prefer H.264(mp4)+AAC(m4a) instead of letting yt-dlp
  pick VP9/Opus, which the device couldn't keep in sync (audio-ahead/video-behind).
  Fixes new MP4 downloads; existing VP9 files would need re-downloading.
- Home shortcuts now open the EXACT Library category: Videos->Videos, Artists->
  Artists, Playlists->Playlists (LibraryScreen gained a jumpTo param).
- Liveliness: finished items in the Downloads list show cover art + a play button.
- Performance: snappier tab transitions (Crossfade 260ms -> 150ms).
